GIT 사용법

* 원격 저장소 동기화

TYPE A
  1. 폴더 생성
  2. git init : git 로컬 저장소 생성
  3. git remote add "url" : 원격 저장소 연결
  4. git pull : 원격 저장소 소스 받기
TYPE B
  1. 폴더 생성
  2. git clone "url" : 원격 저장소 소스 받기

* commit 흐름

  1. 수정 및 작업 후 저장
  2. git pull : commit하기 전 원격 저장소 동기화
  3. git status : 변경된 파일 목록 확인
  4. git diff : 변경된 내용 확인 가능
  5. git add . 또는 git add 파일 이름 : staging area에 추가
  6. git status : staging area에 추가된 파일 목록 확인
  7. git commit -m "메세지" : 작업 관려 메세지와 함께 로컬 저장소에 commit
  8. git log : commit한 history 확인
  9. git push origin ( master 또는 branch명 ) : 원격 저장소에 작업한 내용 동기화

* push 충돌 시 해결 방법

  1. git push 시 reject 메세지와 함께 동기화 실패
  2. git pull(git fetch, git merge를 동시에 실행하는 명령어이기도 함) : 원격 저장소의 소스를 받아온다. auto merge가 실패했다는 메세지가 발생한다.
  3. git status 로 충돌 파일 목록 확인
  4. git diff 또는 cat 파일명으로 충돌 내용 확인
  5. vi 편집기 또는 직접 파일에서 적절히 수정
  6. git status : 변경된 파일 목록 확인
  7. git add 명령어로 staging area에 추가
  8. git status : staging area에 추가된 파일 목록 확인
  9. git commit -m "충돌 해결 메세지" : 로컬저장소에 commit
  10. git log : commit한 history 확인
  11. git push origin ( master 또는 branch명 ) : 원격 저장소에 작업한 내용 동기화

* branch 생성 및 사용

  1. git branch branch이름 : branch 생성
  2. git checkout branch이름 : 해당 branch로 작업 영역 변경 이후 작업 내역은 master영역에는 반영되지 않는다.
  3. 수정 및 작업 후 commit 한다.
  4. git checkout master : master영역으로 작업영역 변경
  5. git merge branch이름 : branch영역에서 작업한 내용을 master영역에 merge한다.
  6. git pull, add, commit, push~

* branch merge 충돌 시 해결 방법

  1. git merge 시 충돌
  2. git status 로 충돌 파일 목록 확인
  3. git diff 또는 cat 파일명으로 충돌 내용 확인
  4. vi 편집기 또는 직접 파일에서 적절히 수정
  5. git status : 변경된 파일 목록 확인
  6. git add 명령어로 staging area에 추가
  7. git status : staging area에 추가된 파일 목록 확인
  8. git commit -m "충돌 해결 메세지" : 로컬저장소에 commit
  9. git log : commit한 history 확인
  10. git push origin ( master 또는 branch명 ) : 원격 저장소에 작업한 내용 동기화

* 파일 되돌리는 방법

  1. git checkout -- : 수정했지만 아직 staging area에 추가하지 않은 파일을 되돌린다.
  2. git reset HEAD : 수정했고 스테이지에 추가한 파일을 다시 staging area에서제거 할 때
  3. git fetch origin, git reset --hard origin/master : 수정했고 로컬저장소에 commit한 파일을 되돌릴 때
